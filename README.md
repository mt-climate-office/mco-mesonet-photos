# Montana Mesonet Photo Explorer

An interactive web map for browsing Montana Mesonet station photos by date, time, and camera
direction. Photos are mirrored from the Mesonet source server to S3, converted to WebP, and
served via CloudFront. The explorer is a static single-page app hosted on GitHub Pages.

A service of the [Montana Climate Office](https://climate.umt.edu).

---

## Components

| Component | Description |
|-----------|-------------|
| `docs/index.html` | Single-page web explorer — D3 Voronoi map, photo grid modal, lightbox, county overlay |
| `scripts/mirror_photos.py` | Incrementally mirrors new photos from the source server to S3, converts to WebP, updates manifest |
| `scripts/backfill_from_local.py` | One-time upload of locally backfilled photos to S3 and manifest rebuild |
| `scripts/uppercase_webp_directions.py` | One-time migration: renames WebP keys in S3 from lowercase to uppercase direction indicators |
| `scripts/generate_preview.py` | Generates `docs/preview.png` via Playwright for `og:image` social preview |
| `terraform/` | AWS infrastructure (S3, CloudFront, IAM/OIDC for GitHub Actions) |
| `.github/workflows/mirror_photos.yml` | Runs `mirror_photos.py` twice daily and regenerates the social preview image |

---

## How It Works

The source server at `https://data.climate.umt.edu/mesonet/photos` hosts station photos named
`YYYYMMDDHHMMSS_DIR.jpg`. `mirror_photos.py` runs a linear six-step pipeline:

1. **Load manifest** — downloads `s3://mco-mesonet/photos/manifest.parquet` to find already-processed photos, and `photos/skip_list.json` for permanently-bad files
2. **Crawl & compare** — crawls the source directory listing for each station and builds a task list of new photos (excludes manifest + skip list entries). Results are cached locally in `cache/crawl_cache.json` to speed up repeated dev runs; use `--fresh-crawl` to force a re-crawl
3. **Download** — fetches raw JPEGs, validating both the SOI (`ff d8`) and EOI (`ff d9`) markers. Truncated or non-JPEG responses are added to the skip list and never retried. Transient network errors are retried on the next run
4. **Convert** — converts each JPEG to WebP via `cwebp` (`-q 75 -m 6 -resize 320 0`)
5. **Upload** — uploads both the raw JPEG and WebP to S3 with immutable cache headers
6. **Update manifest & skip list** — appends new rows to the manifest parquet and, if any new bad files were found, updates `photos/skip_list.json` in S3

The web explorer loads the manifest at startup (via CloudFront), renders station photos as
Voronoi cells on a D3 map, and allows browsing by date, time, and direction. Clicking a cell
opens a photo grid modal; clicking a thumbnail opens a full-size lightbox from `photos/raw/`.

### S3 Bucket Layout

```
s3://mco-mesonet/
└── photos/
    ├── raw/
    │   └── {station}/
    │       └── {YYYY-MM-DDTHH0000}_{DIR}.jpg   # Original full-resolution JPEG
    ├── web/
    │   └── {station}/
    │       └── {YYYY-MM-DDTHH0000}_{DIR}.webp  # 320px-wide WebP (q75, method 6)
    ├── manifest.parquet                         # Snappy-compressed index of all photos
    └── skip_list.json                           # S3 keys for permanently-bad source files
```

The manifest schema:

| Column | Type | Description |
|--------|------|-------------|
| `station` | string | Station ID (e.g. `aceabsar`) |
| `direction` | string | Camera direction (`N`, `S`, `E`, `W`, `SNOW`, `NS`, `SS`) |
| `datetime` | string | `YYYY-MM-DDTHH0000` |
| `s3_key_raw` | string | S3 key for the raw JPEG |
| `s3_key_webp` | string | S3 key for the WebP thumbnail |

## Running Locally

### Docker (recommended)

The `Dockerfile` mirrors the GitHub Actions `ubuntu-24.04` runner exactly (Python 3.12, `cwebp` from apt). Use this to reproduce CI failures locally.

**Build:**

```bash
docker build -t mco-mirror .
```

**Dry run** (crawls source, compares against manifest — no downloads or uploads):

```bash
docker run --rm \
  -v ~/.aws:/root/.aws:ro \
  -v $(pwd)/cache:/repo/cache \
  mco-mirror --dry-run
```

**Full run:**

```bash
docker run --rm \
  -v ~/.aws:/root/.aws:ro \
  -v $(pwd)/cache:/repo/cache \
  mco-mirror --profile mco
```

**Force a fresh crawl** (re-crawl source server instead of using `cache/crawl_cache.json`):

```bash
docker run --rm \
  -v ~/.aws:/root/.aws:ro \
  -v $(pwd)/cache:/repo/cache \
  mco-mirror --profile mco --fresh-crawl
```

**Interactive shell** (for debugging):

```bash
docker run --rm -it \
  -v ~/.aws:/root/.aws:ro \
  -v $(pwd)/cache:/repo/cache \
  --entrypoint bash \
  mco-mirror
```

The `cache/` volume mount persists downloaded files and the crawl cache across container runs.
Local copies of `cache/crawl_cache.json` and `cache/skip_list.json` are written automatically.

### Without Docker

**Prerequisites:** Python 3.12+, `cwebp` (`brew install webp`), AWS profile `mco` configured.

```bash
pip install -r scripts/requirements.txt
python scripts/mirror_photos.py               # full run
python scripts/mirror_photos.py --dry-run     # count new photos only
python scripts/mirror_photos.py --fresh-crawl # force re-crawl of source
```

**Generate social preview image (requires Playwright):**

```bash
pip install playwright && playwright install --with-deps chromium
python scripts/generate_preview.py
```

## CI/CD (GitHub Actions)

The `mirror_photos` workflow runs on manual dispatch (schedule to be re-enabled after backfill
is complete). It:

1. Runs `mirror_photos.py` to pull new photos into S3
2. Invalidates the CloudFront cache for `manifest.parquet`
3. Regenerates `docs/preview.png` via Playwright and commits it back to the repo

**Required secrets:**

| Secret | Description |
|--------|-------------|
| `AWS_ROLE_ARN` | IAM role ARN assumed via OIDC (output by Terraform as `github_actions_role_arn`) |
| `CLOUDFRONT_DISTRIBUTION_ID` | CloudFront distribution ID (output by Terraform) |

---

## Cloud Architecture (AWS)

All infrastructure is managed by Terraform in `terraform/`.

| Resource | Description |
|----------|-------------|
| S3 bucket | `mco-mesonet` — private; photos served exclusively via CloudFront |
| CloudFront distribution | HTTPS CDN with immutable caching for photos, short TTL for manifest |
| IAM OIDC role | Assumed by GitHub Actions; scoped to S3 read/write and CloudFront invalidation |

### Deploy infrastructure

```bash
cd terraform

cp terraform.tfvars.example terraform.tfvars
$EDITOR terraform.tfvars   # fill in aws_account_id and any overrides

terraform init
terraform plan
terraform apply
```

`terraform.tfvars` is gitignored — never commit it.

## Data Source

Station photos are captured by cameras at Montana Mesonet sites and served by the
[Montana Climate Office](https://climate.umt.edu) at
`https://data.climate.umt.edu/mesonet/photos`. Station metadata and photo direction
availability are retrieved from the Mesonet API:
`https://mesonet.climate.umt.edu/api/v2/photos?type=json`.
